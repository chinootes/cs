# Concurrent Mark and Sweep


- Operation
Runs a GC thread alongside the application.

- Used when:
    - More memory available
    - High number of CPUs or cores present
    - App demands short pauses

    (Used in most fintech apps)

(CMS - Concurrent Mark Sweep)
